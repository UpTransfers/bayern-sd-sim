# FC Bayern Sporting Director Simulator

A fan-made, real-data powered sporting director simulator for FC Bayern Munich's 2026–27 planning cycle.

## Overview

You take control of Bayern's transfer planning and squad building. The app lets you:

- review real club and Bundesliga data where free sources expose it
- sell and loan players
- search and sign real players from free public sources
- set a formation and starting XI
- track budget, board confidence, fan pressure, and data confidence
- run a deterministic season projection

## Tech Stack

- Next.js 16
- React 19
- TypeScript strict mode
- Tailwind CSS
- Framer Motion
- Lucide React
- React Hook Form and Zod available for form validation
- Zustand available for client state if extended further
- Supabase Postgres schema included for production deployment

## Setup

```bash
npm install
npm run dev
```

For a production build:

```bash
npm run build
```

## Environment Variables

Copy `.env.example` to `.env.local` and fill in what you need:

- `DATABASE_URL`
- `NEXT_PUBLIC_APP_URL`
- `FOOTBALL_DATA_API_KEY`
- `THESPORTSDB_API_KEY`
- `API_FOOTBALL_KEY`

## Supabase Setup

1. Create a free Supabase project.
2. Run the SQL in [`supabase/schema.sql`](./supabase/schema.sql).
3. Set `DATABASE_URL` to the Supabase connection string.
4. Deploy the app on Vercel or Netlify with the same environment variables.

## API Sources

The app uses the following free or optional sources:

- OpenLigaDB for Bundesliga fixtures, results, and standings
- Wikidata for canonical club and player metadata
- football-data.org if `FOOTBALL_DATA_API_KEY` is set
- TheSportsDB if `THESPORTSDB_API_KEY` is set
- API-Football is reserved as an optional extension if `API_FOOTBALL_KEY` is set

## Data Source Limitations

- Some fields are not available from free sources.
- When a field cannot be verified, the UI shows `Unavailable from free source`.
- Transfer fees generated by the simulator are deterministic estimates unless explicitly sourced from a free API.
- Raw source payloads are stored for auditability.

1. Push the project to GitHub.

3. Configure the environment variables.

5. Deploy.

## Known Limitations

- Free sources may not expose every current player field.
- Some optional sources require API keys.
- Undo is intentionally lightweight in this first pass.
- Production deployments should point `DATABASE_URL` at a persistent Postgres database such as Supabase.
